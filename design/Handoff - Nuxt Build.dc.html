Flip's Kitchen — Nuxt build guide

Component breakdown, the Tailwind theme, and the class recipes to rebuild the prototype in Nuxt 3. Every color, radius and font used in the design is tokenised below so the build stays 1:1 with what you've seen.

01 — Stack & structure

What to reach for

Nuxt 3 (SSR or SPA), Tailwind via @nuxtjs/tailwindcss, fonts via @nuxtjs/google-fonts, a Pinia store (or a useRecipes composable) for recipe data + filters, and vuedraggable (or @vueuse/integrations/useSortable) for the admin reordering.

app/
├─ pages/
│  ├─ index.vue                 → Home (grid + search + filters + states)
│  ├─ recipes/[id].vue          → Recipe detail
│  ├─ recipes/[id]/cook.vue     → Cook mode (full-screen, or a <Teleport> overlay)
│  ├─ recipes/[id]/print.vue    → Print layout (or print CSS on detail)
│  └─ admin/
│     ├─ new.vue                → Create recipe
│     └─ [id]/edit.vue          → Edit recipe

Recipe detail

MetaStrip · IngredientList · MethodSteps · VideoEmbed
Two-column body without media queries: flex flex-wrap gap-8; aside flex-[1_1_280px], method flex-[2.4_1_440px] → stacks on mobile automatically. Ingredient rows are toggle buttons (check state in local ref). Step number badge w-9 h-9 rounded-xl bg-clay-tint text-clay font-display. VideoEmbed: use nuxt-lazy-youtube or a click-to-load facade over an <iframe>.

Cook mode

CookStepper — one step at a time, font-display text-[clamp(28px,4.6vw,46px)], progress bar, big Back / Next buttons, swipe via @vueuse/gesture. CookChecklist — oversized tappable rows, checked state struck through. Toggle between them with a segmented control; expose the default via a prop. Call the Wake Lock API (useWakeLock) on mount so the screen stays on.

Print

PrintSheet is a max-w-[760px] white sheet. Tag chrome class="print:hidden"; the Print button calls window.print(). Add a @media print rule to flatten the sheet (no shadow/border/margins) and set body{background:#fff}.

Admin form

Sectioned cards (rounded-card bg-paper border border-line p-6). IngredientRow / StepRow wrapped in <draggable>; the 6-dot handle is the drag grip; drop target highlights bg-clay-tint border-clay. ImageUploader — <input type=file> + drop zone, preview via URL.createObjectURL. TagInput adds on Enter. DifficultyPicker = 5 clickable dots.
04 — Data & state

The Recipe model + filter store

// types/recipe.ts
export interface Ingredient { qty: string; unit: string; name: string }
export interface Step { text: string; photo?: string | null }
export interface Recipe {
  id: string
  title: string
  cuisine: 'Filipino' | 'Thai' | 'Japanese' | string
  description: string
  prep: number; cook: number; total: number   // minutes
  servings: number
  difficulty: 1 | 2 | 3 | 4 | 5
  tags: string[]
  hero?: string | null
  gallery?: string[]
  youtube?: string | null
  ingredients: Ingredient[]
  steps: Step[]
}
// composables/useRecipeFilters.ts
const query = ref('')
const activeChips = ref<string[]>([])

const filtered = computed(() => recipes.value.filter(r => {
  const q = query.value.trim().toLowerCase()
  const inChips = !activeChips.value.length || activeChips.value.every(c =>
    r.tags.includes(c) || r.ingredients.some(i => i.name.toLowerCase().includes(c.toLowerCase())))
  const inQ = !q || [r.title, r.cuisine, r.tags.join(' ')].join(' ').toLowerCase().includes(q)
    || r.ingredients.some(i => i.name.toLowerCase().includes(q))
  return inChips && inQ
}))

// status drives the Home view: 'loading' | 'empty' | 'no-results' | 'grid'

Chips filter with AND across selections (a selection that matches nothing surfaces the no-results state, which is intended). Switch .every → .some for OR matching if you prefer a broader net.

05 — Responsive & a11y notes
  • Mobile-first throughout. The grid and detail split use auto-fill/minmax and flex-wrap with flex-basis, so they reflow with no breakpoints; add sm: / lg: variants only where you want to force a specific column count.
  • Fluid type via clamp() on the big display headings and cook-mode step text.
  • Tap targets ≥ 44px in cook mode and on all mobile controls. Cook mode nav buttons are full-width and 56px tall.
  • Ingredient / checklist toggles are real <button>s with aria-pressed; the search input needs a visible label or aria-label.
  • The prototype's "State" switcher (top-right on Home) is a dev-only aid to preview loading / empty / no-results — drop it from production.
